# 🔍 Scout 조사 보고서: DC 평가 시뮬레이션 빌더

## 📊 DC(Development Center) 평가 기법 분석

### 1. In-Basket 기법
**개념**: 실제 업무 상황을 모방한 서류함(In-basket) 처리 시뮬레이션
- **구성 요소**:
  - 메모, 보고서, 이메일, 회의 자료 등 다양한 문서
  - 우선순위 설정이 필요한 업무 상황
  - 시간 제약 조건
  - 의사결정 포인트들

- **시나리오 필수 요소**:
  - 배경 설정 (회사/부서/직급)
  - 문서 목록 (10-20개 내외)
  - 각 문서의 중요도/긴급도
  - 처리 시간 배분
  - 평가 기준 (우선순위 판단, 의사결정 품질)

### 2. Role-Playing 기법
**개념**: 특정 역할을 맡아 상호작용하는 시뮬레이션
- **구성 요소**:
  - 역할 설정 (관리자, 부하직원, 고객 등)
  - 갈등 상황 또는 문제 상황
  - 목표와 제약 조건
  - 대화 스크립트 가이드

- **시나리오 필수 요소**:
  - 역할별 배경 정보
  - 상황 설정 및 문제점
  - 상대방 캐릭터 설정
  - 예상 대화 흐름
  - 평가 포인트 (의사소통, 갈등해결, 리더십)

### 3. Presentation 기법
**개념**: 주어진 주제에 대해 발표하는 시뮬레이션
- **구성 요소**:
  - 발표 주제 및 목적
  - 청중 설정
  - 시간 제약
  - 자료 준비 조건

- **시나리오 필수 요소**:
  - 발표 주제 및 배경
  - 청중 특성 (임원진, 동료, 고객 등)
  - 발표 시간 및 Q&A 시간
  - 평가 기준 (논리성, 설득력, 표현력)

### 4. Group Discussion 기법
**개념**: 팀 단위로 문제를 해결하는 토론 시뮬레이션
- **구성 요소**:
  - 토론 주제 및 문제 상황
  - 참가자 역할 분담
  - 시간 제약
  - 합의 목표

- **시나리오 필수 요소**:
  - 토론 주제 및 배경 정보
  - 각 참가자별 입장/이해관계
  - 제약 조건 및 목표
  - 평가 기준 (리더십, 협업, 의사소통)
